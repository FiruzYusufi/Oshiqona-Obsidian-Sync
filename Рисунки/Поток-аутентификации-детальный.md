## 🔧 Компоненты системы аутентификации

### AuthService методы
```typescript
interface AuthService {
  // Основные методы
  login(credentials: LoginCredentials): Observable<AuthResponse>
  register(userData: RegisterData): Observable<RegisterResponse>
  logout(): void
  refreshToken(): Observable<TokenResponse>
  forgotPassword(email: string): Observable<void>
  resetPassword(token: string, password: string): Observable<void>
  
  // Проверки состояния
  isAuthenticated(): boolean
  getCurrentUser(): Observable<User | null>
  getAuthState(): Observable<AuthState>
}
```

### TokenService методы
```typescript
interface TokenService {
  // Управление токенами
  getAccessToken(): string | null
  getRefreshToken(): string | null
  setTokens(accessToken: string, refreshToken: string): void
  removeTokens(): void
  
  // Проверки валидности
  isTokenValid(token?: string): boolean
  isTokenExpired(token?: string): boolean
  getTokenExpirationDate(token?: string): Date | null
  
  // Автоматическое обновление
  refreshToken(): Observable<TokenResponse>
  scheduleTokenRefresh(): void
}
```

### Guards конфигурация
```typescript
// Admin App Routes

// Public Routes

  loadChildren: () => import('./auth/auth.module')
}
```

## 🔒 Безопасность и лучшие практики

### JWT Token Security
- **Access Token**: короткий срок жизни (15 минут)
- **Refresh Token**: длительный срок жизни (7 дней)
- **Secure Storage**: HttpOnly cookies для production
- **Token Rotation**: новый refresh token при каждом обновлении

### Password Security
- **Hashing**: bcrypt с salt
- **Strength Requirements**: минимум 8 символов, цифры, буквы
- **Rate Limiting**: ограничение попыток входа
- **Account Lockout**: блокировка после неудачных попыток

### API Security
- **HTTPS Only**: все запросы через защищённое соединение
- **CORS Configuration**: настройка разрешённых доменов
- **Request Validation**: валидация всех входящих данных
- **Error Handling**: не раскрывать внутреннюю информацию
