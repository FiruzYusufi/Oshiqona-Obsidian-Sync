### 2️⃣ Процесс свайпа (Like/Dislike)

```
┌─────────────────────────────────────────────────────────────┐
│                    SWIPE PROCESS                            │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  👤 User                                                    │
│   │                                                         │
│   ├── 1. Просматривает карточку пользователя               │
│   │                                                         │
│   ├── 2. Принимает решение                                 │
│   │    ├── Свайп вправо (Like) 👍                          │
│   │    ├── Свайп влево (Dislike) 👎                        │
│   │    ├── Нажатие кнопки "❤️"                             │
│   │    └── Нажатие кнопки "✖️"                             │
│   │                                                         │
│   └── 3. Выполняет действие                                │
│        │                                                   │
│        ▼                                                   │
│  🎯 HomeComponent                                          │
│   │                                                         │
│   ├── 4. onSwipe(direction: 'left' | 'right')              │
│   │                                                         │
│   ├── 5. Анимация карточки                                 │
│   │    ├── Поворот и перемещение                           │
│   │    ├── Изменение прозрачности                          │
│   │    └── Показ индикатора (❤️ или ✖️)                    │
│   │                                                         │
│   ├── 6. Определение типа действия                         │
│   │    ├── direction === 'right' → Like                    │
│   │    └── direction === 'left' → Dislike                  │
│   │                                                         │
│   └── 7. swipeService.performSwipe(targetUserId, action)   │
│        │                                                   │
│        ▼                                                   │
│  🔧 SwipeService (@core)                                   │
│   │                                                         │
│   ├── 8. Создание объекта свайпа                           │
│   │    ├── fromUserId = currentUser.id                     │
│   │    ├── toUserId = targetUser.id                        │
│   │    ├── action = 'like' | 'dislike'                     │
│   │    └── timestamp = new Date()                          │
│   │                                                         │
│   ├── 9. Оптимистичное обновление UI                       │
│   │    └── Удаление карточки из локального массива         │
│   │                                                         │
│   ├── 10. HTTP POST /api/swipes                            │
│   │     └── Body: { fromUserId, toUserId, action }         │
│   │                                                         │
│   └── 11. Обработка ответа                                 │
│        │                                                   │
│        ▼                                                   │
│  🌐 API Backend (Match Engine)                             │
│   │                                                         │
│   ├── 12. Сохранение свайпа в базе данных                  │
│   │     └── INSERT INTO Swipes (...)                       │
│   │                                                         │
│   ├── 13. Проверка на взаимный лайк (только для Like)      │
│   │     └── SELECT * FROM Swipes WHERE                     │
│   │         fromUserId = toUserId AND toUserId = fromUserId │
│   │         AND action = 'like'                            │
│   │                                                         │
│   ├── 14. Если взаимный лайк найден                        │
│   │     ├── Создание нового чата                           │
│   │     │   └── INSERT INTO Chats (user1Id, user2Id)       │
│   │     │                                                  │
│   │     ├── Отправка уведомлений обеим сторонам            │
│   │     │   └── NotificationHub.SendMatchNotification()    │
│   │     │                                                  │
│   │     └── Возврат информации о матче                     │
│   │         └── { isMatch: true, chatId, matchedUser }     │
│   │                                                         │
│   └── 15. Если обычный свайп                               │
│         └── { isMatch: false, swipeId }                    │
│              │                                             │
│              ▼                                             │
│  🔧 SwipeService (Response Handling)                       │
│   │                                                         │
│   ├── 16. Обработка результата                             │
│   │                                                         │
│   ├── 17. Если матч найден (isMatch: true)                 │
│   │     ├── Показ модального окна "It's a Match!"          │
│   │     ├── Воспроизведение звука/анимации                 │
│   │     ├── Добавление чата в список                       │
│   │     └── Опция "Send Message" или "Keep Swiping"        │
│   │                                                         │
│   └── 18. Эмит события swipeCompleted                      │
│        │                                                   │
│        ▼                                                   │
│  🎯 HomeComponent (Next Card)                              │
│   │                                                         │
│   ├── 19. Переход к следующей карточке                     │
│   │                                                         │
│   ├── 20. Если карточки закончились                        │
│   │     ├── Показ сообщения "No more cards"                │
│   │     ├── Кнопка "Expand search radius"                  │
│   │     └── Кнопка "Refresh recommendations"               │
│   │                                                         │
│   └── 21. Предзагрузка следующих рекомендаций              │
│         └── recommendationService.loadMore()               │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

### 3️⃣ Алгоритм совместимости (ML Engine)

```
┌─────────────────────────────────────────────────────────────┐
│                COMPATIBILITY ALGORITHM                      │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  🧠 Machine Learning Engine                                 │
│   │                                                         │
│   ├── 1. Сбор данных пользователя                          │
│   │    ├── Профиль: возраст, местоположение, образование   │
│   │    ├── Интересы: хобби, музыка, спорт, фильмы          │
│   │    ├── Предпочтения: возрастной диапазон, расстояние   │
│   │    ├── Поведение: время активности, частота свайпов    │
│   │    └── История: предыдущие лайки, матчи, сообщения     │
│   │                                                         │
│   ├── 2. Векторизация признаков                            │
│   │    ├── Категориальные → One-hot encoding               │
│   │    ├── Численные → Нормализация (0-1)                  │
│   │    ├── Текстовые → TF-IDF векторы                      │
│   │    └── Геолокация → Координаты + расстояние            │
│   │                                                         │
│   ├── 3. Расчёт совместимости по компонентам               │
│   │                                                         │
│   │    🎯 Интересы (30% веса)                              │
│   │    ├── Jaccard similarity для общих интересов          │
│   │    ├── Cosine similarity для векторов предпочтений     │
│   │    └── Бонус за редкие общие интересы                  │
│   │                                                         │
│   │    📍 Географическая близость (25% веса)               │
│   │    ├── Расстояние в км (обратная зависимость)          │
│   │    ├── Один город = +0.3 бонус                         │
│   │    └── Разные страны = -0.2 штраф                      │
│   │                                                         │
│   │    👥 Возрастная совместимость (20% веса)              │
│   │    ├── Разница в возрасте (оптимум: 2-5 лет)           │
│   │    ├── Соответствие предпочтениям                      │
│   │    └── Социальная приемлемость                         │
│   │                                                         │
│   │    ⚡ Активность пользователя (15% веса)                │
│   │    ├── Частота входов в приложение                     │
│   │    ├── Время последней активности                      │
│   │    ├── Количество загруженных фото                     │
│   │    └── Заполненность профиля                           │
│   │                                                         │
│   │    🎭 Поведенческие паттерны (10% веса)                │
│   │    ├── Селективность свайпов (like rate)               │
│   │    ├── Время, проводимое на карточке                   │
│   │    ├── Паттерны активности (время дня)                 │
│   │    └── Отзывчивость в чатах                            │
│   │                                                         │
│   ├── 4. Агрегация оценок                                  │
│   │    └── FinalScore = Σ(ComponentScore × Weight)          │
│   │                                                         │
│   ├── 5. Применение машинного обучения                     │
│   │    ├── Collaborative Filtering                         │
│   │    │   └── "Пользователи с похожими вкусами"           │
│   │    │                                                   │
│   │    ├── Content-Based Filtering                         │
│   │    │   └── "Похожие профили и интересы"                │
│   │    │                                                   │
│   │    └── Hybrid Approach                                 │
│   │        └── Комбинация обоих методов                    │
│   │                                                         │
│   ├── 6. Обучение на обратной связи                        │
│   │    ├── Лайки → положительная обратная связь            │
│   │    ├── Дизлайки → отрицательная обратная связь         │
│   │    ├── Матчи → сильная положительная связь             │
│   │    ├── Сообщения → очень сильная положительная связь   │
│   │    └── Блокировки → сильная отрицательная связь        │
│   │                                                         │
│   └── 7. Персонализация алгоритма                          │
│        ├── Адаптация весов под пользователя                │
│        ├── A/B тестирование разных подходов                │
│        └── Непрерывное обучение модели                     │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

  // Основные методы
  getRecommendations(filters?: FilterOptions): Observable<UserCard[]>
  loadMoreRecommendations(): Observable<UserCard[]>
  refreshRecommendations(): Observable<UserCard[]>

  getRecommendationStats(): RecommendationStats

interface CompatibilityEngine {
  // Расчёт совместимости

## 📊 Метрики и аналитика

### Ключевые показатели
- **Match Rate**: процент взаимных лайков
- **Swipe Rate**: количество свайпов на пользователя
- **Engagement Rate**: активность после матча
- **Retention Rate**: возвращение пользователей
- **Recommendation Accuracy**: точность алгоритма

### A/B тестирование
- **Алгоритмы совместимости**: разные веса компонентов
- **UI/UX**: различные дизайны карточек
- **Фильтры**: влияние на качество матчей
- **Timing**: оптимальное время показа рекомендаций

### Машинное обучение
- **Collaborative Filtering**: на основе поведения похожих пользователей
- **Content-Based**: на основе характеристик профилей
- **Deep Learning**: нейронные сети для сложных паттернов
- **Reinforcement Learning**: оптимизация на основе обратной связи
