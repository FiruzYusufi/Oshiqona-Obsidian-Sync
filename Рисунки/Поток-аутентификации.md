# Поток аутентификации

## Диаграмма входа

```
Пользователь
    │
    ▼
[Открывает /auth/sign-in]
    │
    ▼
[alreadyAuthGuard]
    │
    ├── Уже авторизован? → Редирект на /matches
    │
    └── Не авторизован? → Показать форму входа
            │
            ▼
        [Ввод email + пароль]
            │
            ▼
        [AuthService.signIn()]
            │
            ▼
        [POST /api/auth/sign-in]
            │
            ├── Ошибка → Показать сообщение об ошибке
            │
            └── Успех → TokenService.setToken(accessToken)
                            │
                            ▼
                        [Редирект на /matches]
```

## Диаграмма защиты маршрутов

```
Пользователь переходит на /matches
    │
    ▼
[clientAuthGuard]
    │
    ├── Нет токена? → Редирект на /auth/sign-in
    │
    ├── Токен истёк? → authInterceptor.refreshToken()
    │                       │
    │                       ├── Успех → Продолжить
    │                       └── Ошибка → Редирект на /auth/sign-in
    │
    └── Токен валиден? → Показать страницу
```

## Диаграмма обновления токена

```
HTTP Request
    │
    ▼
[authInterceptor]
    │
    ▼
[Добавить Bearer token в заголовок]
    │
    ▼
[Отправить запрос]
    │
    ├── 200 OK → Вернуть ответ
    │
    └── 401 Unauthorized
            │
            ▼
        [TokenService.refreshToken()]
            │
            ├── Успех → Повторить запрос с новым токеном
            │
            └── Ошибка → AuthService.signOut() → /auth/sign-in
```
