# Архитектурные решения (ADR)

## ADR-001: Nx Monorepo

**Дата**: 2026-01-01  
**Статус**: ✅ Принято

### Контекст
Нужно управлять двумя приложениями (Admin + Client) с общим кодом.

### Решение
Использовать **Nx Monorepo** с разделением на `apps/` и `libs/`.

### Преимущества
- Единая кодовая база
- Переиспользование библиотек
- Единые инструменты (lint, test, build)
- Кэширование сборки

### Недостатки
- Сложнее настройка
- Большой репозиторий

## ADR-003: SignalR для реал-тайм

**Дата**: 2026-02-01  
**Статус**: ✅ Принято

### Контекст
Нужна реал-тайм коммуникация для чатов.

### Решение
**Microsoft SignalR** с WebSocket транспортом.

### Преимущества
- Автоматическое переподключение
- Fallback транспорты (SSE, Long Polling)
- Хорошая интеграция с .NET backend
- Поддержка групп и хабов

### Альтернативы
- Socket.io — не подходит для .NET
- WebSocket напрямую — нет автопереподключения

## ADR-005: JWT + LocalStorage

**Дата**: 2026-02-15  
**Статус**: ✅ Принято (с оговорками)

### Контекст
Хранение токенов аутентификации.

### Решение
JWT токены в **LocalStorage**.

### Преимущества
- Простота реализации
- Доступность из JavaScript

### Недостатки
- Уязвимость к XSS
- Рекомендуется перейти на HttpOnly cookies

### Планируемое улучшение
Перейти на HttpOnly cookies в версии 1.0.

## ADR-007: OnPush Change Detection

**Дата**: 2026-03-10  
**Статус**: ✅ Принято

### Контекст
Оптимизация производительности Angular.

### Решение
Использовать **ChangeDetectionStrategy.OnPush** для всех компонентов.

### Преимущества
- Меньше проверок изменений
- Лучшая производительность
- Предсказуемое поведение

### Требования
- Использовать Observables и async pipe
- Иммутабельные данные
- Явный markForCheck() при необходимости
